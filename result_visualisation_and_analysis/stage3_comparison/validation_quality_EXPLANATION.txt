FIGURE DESCRIPTION
--------------------------------------------------------------------------------
This figure compares the validation performance between ontology-guided and
baseline methods in Stage 3 (validation phase).

LEFT PANEL: Ontology Schema Compliance Rate
  - Shows the % of validation rules passed (out of 7 critical rules)
  - Green bars: Ontology-Guided method
  - Orange bars: Baseline method
  - Green dashed line: 100% pass rate (all rules passed)

RIGHT PANEL: Entity Semantic Accuracy
  - Shows the % of entities with semantically correct type assignments (LLM-validated)
  - Same color scheme as ontology schema compliance rate panel


METRIC DEFINITIONS
--------------------------------------------------------------------------------
Ontology Schema Compliance Rate = (Critical Rules Passed / Total Critical Rules) × 100%
  - Total Critical Rules: 7 (VR001-VR005, VR009-VR010)
  - Measures: Structural schema compliance
  - 100% = All validation rules passed (perfect structural quality)

Entity Semantic Accuracy = (Correctly Typed Entities / Total Entities) × 100%
  - Measures: Semantic correctness of entity type assignments using LLM validation
  - 100% = All entities semantically correct
  - <100% = Some entities have incorrect type assignments

INTERPRETATION
--------------------------------------------------------------------------------
The ontology-guided method's superior schema compliance rate demonstrates the
effectiveness of ontology-constrained extraction. By enforcing schema compliance
during extraction, the method produces entities that pass more validation rules.

The baseline method's lower compliance rate and retention rate reflect its permissive
extraction approach. While it extracts more entities in Stage 2, many violate
ontological constraints (e.g., invalid entity types, missing required properties,
incompatible relationships) and fail critical validation rules.

This validates the core hypothesis: ontology-guided extraction not only reduces
computational cost but also improves the quality and usability of the extracted
knowledge graph by preventing invalid entities from entering the pipeline.
